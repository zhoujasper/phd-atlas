# PhD Atlas 公共版路线图

[English](TODO.md) | [简体中文](TODO.zh-CN.md)

公共版首先要成为可靠的个人/自托管工作空间。以下项目按大致交付顺序排列。

## 结束 Beta 的条件

- [ ] 冻结第一个稳定公共版的数据库结构和迁移契约。
- [ ] 建立有版本号、有自动化测试的数据库迁移，并明确稳定公共版本之间的升级兼容保证。
- [ ] 发布稳定版数据支持政策。Beta 版本暂不保证数据库结构或已存数据的向后兼容。

## 产品

- [ ] 将 Team/机构协作模型带到公共版，包括所有者、教师、学生角色，完整邀请生命周期，
      权限测试、审计恢复，以及从个人工作空间安全迁移。
- [ ] 继续完整的手机端适配：小屏布局、触摸优先的上下文操作、用底部弹层替换密集菜单、
      长列表虚拟化，以及在 320–480 px 真机浏览器中验证动画。
- [ ] 改进平板分栏行为和可安装 PWA 的导航体验。

## 数据层

- [ ] 在现有 SQLite 仓储层外建立正式的存储适配器接口。
- [ ] 增加可用于生产的 MySQL 适配器，包括迁移、事务测试、备份/恢复文档，
      以及 SQLite 到 MySQL 的导入工具。
- [ ] 在适配器契约和 MySQL 实现稳定后评估 PostgreSQL。

## 运维

- [ ] 为带标签的版本发布签名容器镜像。
- [x] 自动生成带清单校验的 GitHub Release 更新包，并在发布前测试安装和回滚路径。
- [ ] 将原生更新冒烟测试扩展到托管 Ubuntu、CentOS Stream/RHEL 兼容 Linux
      和 Windows Server runner。
- [ ] 为上传文件和备份增加可选对象存储支持。

## 集成与质量

- [ ] 扩充与供应商无关的 AI 配置和本地模型文档。
- [ ] 增加更多日历和邮件服务商配置示例。
- [ ] 每次公共功能上线时持续维护键盘、屏幕阅读器、减少动态效果、高对比度和多语言回归测试。
